# 元件測試資料夾說明

這份文件說明 `元件測試` 裡面所有資料夾的用途。這個資料夾主要放各種元件測試與整合版本，包含毫米波雷達、紅外線熱感測器、WiFi TCP 傳輸、語音播放、按鈕與 GPIO 狀態控制。

## 總覽

`元件測試` 裡的資料夾大多是 Arduino sketch。每個 `sketch_*` 資料夾通常包含：

- 一個 `.ino` 主程式。
- 有些資料夾會有 `.txt` 備註，說明該版本測試內容。

## 資料夾內容

### `sketch_nov12a`

毫米波雷達跌倒偵測測試。

內容：

- 使用 UART 與毫米波雷達通訊。
- 傳送雷達初始化指令。
- 開啟跌倒偵測。
- 設定靈敏度。
- 設定跌倒判斷時間。
- 設定安裝高度。
- 解析雷達回傳封包，判斷是否跌倒。

用途：

- 測試毫米波雷達是否能成功偵測跌倒。

備註檔：

- `毫米波雷達成功測到跌倒.txt`

### `sketch_nov18b`

老人跌倒偵測 1.0 整合版。

內容：

- 整合 MR60FDA1 毫米波跌倒偵測雷達。
- 整合 BMS26M833 / AMG8833 紅外線熱感測器。
- 整合 BMV31T001 語音播放模組。
- 程式中有 TCP 傳輸邏輯，可把資料送出。

用途：

- 老人跌倒偵測專題的早期整合版本。
- 同時測試雷達、紅外線感測與語音播放。

備註檔：

- `老人跌倒1.0.txt`

### `sketch_nov25a`

紅外線偵測人類測試。

內容：

- 使用 `BMS26M833` 函式庫。
- 讀取 AMG8833 / BMS26M833 的 8x8 溫度矩陣。
- 讀取環境溫度。
- 用溫度差判斷是否有人。
- 將 64 個溫度點印到序列埠監控視窗。

用途：

- 測試紅外線熱感測器是否能偵測人體熱源。

備註檔：

- `紅外線偵測人類.txt`

### `sketch_nov25b`

紅外線與毫米波雷達整合測試。

內容：

- 使用毫米波雷達做跌倒偵測。
- 使用紅外線熱感測器做人體熱源判斷。
- 雷達與紅外線兩種感測資料一起使用。
- 偵測到跌倒後，程式會改變後續流程。

用途：

- 把跌倒偵測與人體熱源偵測合併。
- 測試兩種感測器整合後是否能正常運作。

備註檔：

- `紅外線和毫米波雷達整合.txt`

### `sketch_nov30a`

跌倒偵測加上停留時間測試。

內容：

- 使用毫米波雷達做跌倒偵測。
- 開啟人體存在偵測。
- 設定停留時間，例如 60 秒。
- 解析雷達封包，判斷跌倒、靜止或停留時間過長。

用途：

- 在原本跌倒偵測功能上，增加停留時間 / 靜止時間判斷。

備註檔：

- `在原有的跌倒上增設定停留時間.txt`

### `sketch_dec25d`

雷達封包處理片段。

內容：

- 處理雷達 UART 封包。
- 判斷跌倒封包。
- 判斷人體動作封包。
- 偵測到跌倒時送 `1` 到 server。
- 偵測到人體動作時送 `[7,數值]` 到 server。

用途：

- 測試或暫存雷達封包處理邏輯。

注意：

- 這個檔案看起來不是完整 Arduino sketch，因為開頭就是函式，可能需要搭配其他程式碼才能編譯。

### `sketch_dec25e`

WiFi + 毫米波雷達 + 按鈕狀態整合版。

內容：

- WiFi 使用 UART0。
- 毫米波雷達使用 UART1。
- 透過 AT 指令連接 WiFi。
- 連線到 TCP server `10.77.252.61:8000`。
- 偵測跌倒後送資料到 server。
- 偵測人體動作後送 movement value。
- D7 按鈕切換狀態。
- D6 / D5 依照狀態輸出。
- `stateCount` 會影響目標秒數，例如 8 小時、6 小時、5 秒。

用途：

- 測試雷達偵測結果透過 WiFi 傳送。
- 測試按鈕切換模式與輸出腳位。

### `sketch_dec26c`

WiFi + 毫米波雷達 + AMG8833 紅外線熱感測器整合版。

內容：

- WiFi 連到 TCP server `10.77.252.61:8000`。
- 毫米波雷達透過 UART1 接收資料。
- 用 `Wire` 直接讀 AMG8833 的 I2C 暫存器。
- 讀取 64 個紅外線溫度點。
- 保留按鈕 D7 與 D6 / D5 狀態輸出。

用途：

- 把 WiFi、毫米波雷達、紅外線熱感測器整合在同一支程式。
- 適合測試完整感測資料與 server 傳輸。

### `sketch_dec27a`

WiFi + 毫米波雷達 + 活動歷史紀錄版本。

內容：

- WiFi 連到 TCP server `10.77.252.61:8000`。
- 毫米波雷達透過 UART1 接收資料。
- 偵測跌倒與人體動作。
- 使用 `activityHistory` 記錄一段時間內的活動狀態。
- 有 D7 按鈕切換狀態。
- D6 / D5 依照狀態輸出。

用途：

- 在 WiFi + 雷達整合基礎上，增加活動歷史紀錄。
- 可用來判斷長時間活動狀態或不活動狀態。

## 簡單分類

- 毫米波雷達 / 跌倒偵測：`sketch_nov12a`、`sketch_nov25b`、`sketch_nov30a`、`sketch_dec25d`、`sketch_dec25e`、`sketch_dec26c`、`sketch_dec27a`
- 紅外線熱感測器：`sketch_nov18b`、`sketch_nov25a`、`sketch_nov25b`、`sketch_dec26c`
- WiFi / TCP 傳輸：`sketch_nov18b`、`sketch_dec25e`、`sketch_dec26c`、`sketch_dec27a`
- 語音播放：`sketch_nov18b`
- 按鈕 / GPIO 狀態控制：`sketch_dec25e`、`sketch_dec26c`、`sketch_dec27a`

## 建議查看順序

如果要了解專題功能演進，可以照這個順序看：

1. `sketch_nov12a`：先看毫米波雷達如何偵測跌倒。
2. `sketch_nov25a`：再看紅外線如何偵測人體熱源。
3. `sketch_nov25b`：看雷達與紅外線如何整合。
4. `sketch_nov30a`：看如何增加停留時間判斷。
5. `sketch_dec25e`：看 WiFi、雷達、按鈕狀態如何整合。
6. `sketch_dec26c`：看 WiFi、雷達、紅外線熱感測器完整整合。
7. `sketch_dec27a`：看活動歷史紀錄版本。
