一、總覽

本資料夾實際名稱為「BMC81M001 Arduino library V1.0.5」，不是小寫的 bmc81m001。
它是一個 Arduino library，核心用途是讓 Arduino 透過 UART 與 BMC81M001 WiFi 模組通訊，並用 AT command 包裝 WiFi 連線、TCP、MQTT、HTTP GET、模組狀態查詢等功能。

資料夾內共有 26 個檔案：
- 1 個 Arduino library metadata：library.properties
- 2 個文件：README.md、LICENSE.md
- 1 個 Arduino IDE 關鍵字設定：keywords.txt
- 2 個 library 原始碼：BMC81M001.h、BMC81M001.cpp
- 1 個主要 Arduino 範例：TCP.ino
- 2 個範例輔助 header：TCP.h、voice_cmd_list.h
- 14 個歷史版本/實驗記錄程式：examples/TCP/記錄/*.cpp
- 2 個圖片：layout.png、schematic.png
- 1 個 Windows 縮圖快取：Thumbs.db

注意：
- examples/TCP/TCP.h 內含 WiFi SSID、WiFi 密碼、TCP Server IP 與 Port，若要公開或交給別人，建議先移除或改成占位文字。
- 多個 .ino/.cpp 檔案中的中文註解在目前讀取環境下出現亂碼，推測可能是原始檔案編碼與 PowerShell 顯示/讀取編碼不一致。程式邏輯仍可由英文符號、函式與常數判讀。

6. BMC81M001/src/BMC81M001.cpp

行數：1086
大小：31178 bytes

Library 的主要實作檔，負責把高階函式轉成 AT command 並透過 UART 送給 BMC81M001/ESP 類型模組。

主要功能分類：

A. 建構與初始化
- BMC81M001(HardwareSerial *theSerial)
  使用硬體序列埠。
- BMC81M001(uint16_t rxPin, uint16_t txPin)
  建立 SoftwareSerial。
- begin(uint32_t baud)
  初始化硬體或軟體序列埠。

B. WiFi/TCP
- connectToAP(ssid, pass)
  送 AT+CWMODE=1 設為 station mode，再送 AT+CWJAP 連線 AP。
- connectTCP(ip, port)
  送 AT+CIPSTART 建立 TCP 連線。
- writeDataTcp(Dlength, Dbuffer)
  先送 AT+CIPSEND=長度，再送資料內容。
- readDataTcp()
  讀 UART buffer，尋找 +IPD，再取冒號後面的 TCP payload。

C. MQTT
- configMqtt(clientid, username, password, mqtt_host, server_port)
  依序設定 MQTT usercfg、clientid、username、password，最後 AT+MQTTCONN 連線。
- setSubscribetopic(subscribetopic)
  送 AT+MQTTSUB 訂閱 topic。
- writeString(Dbuffer, topic)
  送 AT+MQTTPUB 發布字串 payload。
- writeBytes(Dbuffer, Dlength, topic)
  送 AT+MQTTPUBRAW 發布 raw bytes。
- readIotData(...)
  解析 +MQTTSUBRECV，取出 topic、payload 長度與 payload。

D. HTTP GET
- http_begin(serverURL, port, subURL)
  解析 http:// 或 https://，設定 TCP/SSL、host、port、path 與 request 長度。
- http_get()
  建立連線、送 GET request、讀回 HTTP response body。
- http_getString()
  將 BMC81M001Response 轉成 String。
- http_end()
  送 AT+CIPCLOSE 關閉連線。

E. 基本 AT command 與狀態查詢
- reset()
  送 AT+RST。
- sendATCommand(...)
  送 AT command，等待回覆包含 OK。
- sendATCmd(...)
  類似 sendATCommand，但回傳完整 ACK 字串或 TimeOut。
- SSID()
  送 AT+CWLAP 掃描附近 WiFi。
- getStatus()
  送 AT+CIPSTATUS 並解析狀態碼。
- getSSID()
  送 AT+CWJAP? 取得目前連線 SSID。
- getIP()/getGateway()/getMask()
  送 AT+CIPSTA? 並用 strtok 解析 IP/Gateway/Mask。
- getMacAddress()
  送 AT+CIPSTAMAC?，解析並轉成大寫 MAC 字串。
- getATVersion()
  送 AT+GMR，取版本字串。

F. Buffer 操作
- readResponse()
  從 Serial 或 SoftwareSerial 讀入 BMC81M001Response。
- clearResponse()
  清空回應 buffer 並重設 resLength。

掃描時看到的程式注意事項：
- http_get() 內部計算了 result，但最後固定 return HTTP_GET_OP_SUCCESS，代表 timeout 或 COMMUNICAT_ERROR 可能不會正確回傳給呼叫端。
- http_get() 的接收 loop 直接使用 _serial->available()，如果物件是 SoftwareSerial 建立，這段可能有空指標風險。
- sendATCommand() 裡有一行 String cmd = "AT+MQTTPUB=0,\""; 但 cmd 沒被使用，屬於殘留變數。
- getATVersion() 使用 char sourceStr[20] 與 memcpy，但未明確補 '\0'，轉成 String 時可能有尾端資料風險。
- 多處使用固定長度 char buffer，若 AT 回應格式變動或資料過長，要留意溢位與解析失敗。

8. BMC81M001/examples/TCP/TCP.ino

行數：411
大小：13941 bytes

目前 TCP 範例的主 Arduino sketch。內容已不只是單純 BMC81M001 TCP 範例，而是整合三個硬體模組的應用：
- BMC81M001 WiFi module：透過 Serial 與 TCP server 通訊。
- BMS26M833/AMG8833：讀取 8x8 熱成像陣列與 thermistor 溫度。
- MR60FDA1 類雷達模組：透過 Serial4 讀取跌倒偵測與人體活動量封包。
- BMV31T001 語音模組：播放警告音檔。

setup() 流程：
1. Serial 與 Serial4 設為 115200。
2. 初始化 BMC81M001。
3. 連接 WiFi AP。
4. 建立 TCP 連線。
5. 對雷達送出設定指令：
   queryCmd、enableCmd、sensitivityCmd、fallTimeCmd、heightCmd。
6. 初始化 AMG8833。
7. 初始化 BMV31T001 語音模組、開電源、設定音量、初始化音源。
8. 將 activityHistory 初始化為 100。

loop() 流程：
1. 呼叫 Wifi.readDataTcp() 接收 server 訊息。
2. 使用 parseServerNumber() 從 server 訊息解析數字，例如 [10] 或 10。
3. 若解析成功，更新 serverTargetSeconds，並重設活動量歷史資料。
4. 持續從 Serial4 讀雷達封包並交給 processIncomingByte()。
5. 讀取 AMG8833 pixels 與 thermistor 溫度。
6. 以 TCP 傳送 THERM=xx.xx 給 server。
7. delay 500 ms。

雷達封包處理：
- 封包 header：0x53 0x59。
- 封包 tail：0x54 0x43。
- processIncomingByte() 以狀態機累積封包。
- isFallPacket() 判斷跌倒事件。
- isBodyMovementPacket() 判斷 body movement 封包。

事件行為：
- 偵測跌倒：
  - TCP 傳送 "1"。
  - 播放 voice_table[0]。
- 偵測活動量：
  - buf[6] 作為 movementValue。
  - 寫入 activityHistory ring buffer。
  - 若 serverTargetSeconds > 0，檢查最近 N 筆是否連續低於 5。
  - 若連續靜止達標，播放 voice_table[1]，TCP 傳送 "8"，重設活動量歷史。
  - 每次活動量封包會傳送 "[7,movementValue]"。
- parseServerNumber() 成功解析 server 指定秒數後，會 TCP 回傳 "over"。

注意：
- 此檔案與 記錄/1218_1.cpp 的大小與行數相同，判斷 TCP.ino 很可能是 1218_1.cpp 的目前正式版或複製版。
- 目前檔案中文註解顯示有大量亂碼，建議確認原始編碼後轉成 UTF-8。
- 這個範例依賴外部 library：BMS26M833、BMV31T001，以及可用的 Serial4 板子。一般 UNO 沒有 Serial4，需使用支援多硬體 UART 的開發板。

六、目前應用程式的整體功能判讀

從 TCP.ino 與記錄資料夾可看出，這個專案目前已經從單純 BMC81M001 WiFi library 範例，演變成一個「跌倒/靜止偵測 + 溫度上傳 + TCP server 控制 + 語音提醒」的整合應用。

資料流大致如下：

1. Arduino 開機後連 WiFi。
2. 連到 TCP Server：10.81.63.61:8000。
3. 對雷達模組送設定指令，啟用跌倒偵測、設定靈敏度、跌倒時間、高度。
4. 讀雷達 Serial4 封包。
5. 如果是跌倒封包：
   - 傳 "1" 給 server。
   - 播放第 1 個語音。
6. 如果是活動量封包：
   - 取 buf[6] 當 movementValue。
   - 傳 "[7,movementValue]" 給 server。
   - 存入最近 200 筆歷史。
   - 如果 server 指定 N 秒，且最近 N 筆都低於 5，視為長時間靜止。
   - 長時間靜止時播放第 2 個語音並傳 "8" 給 server。
7. 讀 AMG8833 thermistor。
8. 傳 "THERM=xx.xx" 給 server。
9. 若 server 傳來 N 或 [N]：
   - 解析成 serverTargetSeconds。
   - 重設活動量歷史。
   - 回傳 "over" 表示收到設定。
