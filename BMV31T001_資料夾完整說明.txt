一、總覽

本資料夾實際 library 根目錄為：
BMV31T001/BMV31T001

這是一個 Arduino library，用來控制 Best Modules 的 BMV31T001 Voice Playback Shield。BMV31T001 主要透過單線 one-wire 類型的資料腳位控制語音播放，也支援按鍵、LED、電源控制，以及透過序列埠/燒錄流程更新語音來源。

資料夾內共有 102 個檔案：
- 4 個 library 文件/metadata：README.md、LICENSE.md、library.properties、keywords.txt
- 2 個 library 原始碼：src/BMV31T001.h、src/BMV31T001.cpp
- 1 個 Arduino 範例 sketch：voiceUpdateAndPlayback.ino
- 2 個 voice command list：範例根目錄與 VoiceBroadcast 專案各一份
- 2 張接線/電路圖片：layout.PNG、schematic.PNG
- 7 個 VoiceBroadcast 專案設定/輸出檔：Demo_key_mapping.h、PrjSetting.ini、set.ini、Setting.ini、VoiceBroadcast.dat、VoiceBroadcast.ini、VoiceBroadcast.vup
- 70 個 Voice Files 檔案：10 組語音，每組 mp3/wav/RAW/raw2/PCM/COD/FREQ.TXT 等格式
- 10 個 output wav 檔案

注意：
- 第 10 組語音在設定檔中顯示為「十.mp3 / 十.wav」，但在目前檔案系統讀取結果中顯示為「坋.*」。這很可能是中文檔名編碼顯示問題。
- 多個 source 註解中的中文或特殊符號有亂碼，推測是原始檔案編碼與目前讀取環境不一致。
- BMV31T001.cpp 內 pin 腳位是寫死的，例如 DATA=12、LED=8、STATUS=9、POWER=A0、按鍵 A1-A5；若硬體不同需修改 library source 或封裝成可設定參數。

6. src/BMV31T001.cpp

行數：1335
大小：42674 bytes

BMV31T001 的主要實作檔。

硬體 pin 定義：
- KEY_UP = A1
- KEY_LEFT = A2
- KEY_DOWN = A3
- KEY_RIGHT = A4
- KEY_MIDDLE = A5
- POWER_PIN = A0
- LED_PIN = 8
- STATUS_PIN = 9
- DATA = 12
- SPI chip select SEL = 10
- ICPCK = 13
- ICPDA = 11

播放控制 command：
- PAUSE_PLAY = 0xF1
- CONTINUE_PLAY = 0xF2
- LOOP_PLAY = 0xF4
- STOP_PLAY = 0xF8

主要功能分類：

A. 初始化
- BMV31T001()
  初始化 key、update address 等內部狀態。
- begin()
  reset 模組，設定 POWER/LED/DATA/ICPCK/STATUS/KEY pins，並 delay 1000 ms 等待模組就緒。

B. 單線播放控制
- writeCmd(cmd, data)
  用 DATA pin 送出 command。資料 bit 以不同 high/low pulse 長度表示 1/0：
  bit 1：HIGH 約 1200 us、LOW 約 400 us
  bit 0：HIGH 約 400 us、LOW 約 1200 us
  每個 command 前有 start signal。
- setVolume(volume)
  送 0xE1 + volume。volume 0 到 11。
- playVoice(num, loop)
  num < 128 時送 0xFA + num；num >= 128 時送 0xFB + num % 128。
  loop 為 true 時加送 0xF4。
- playSentence(num, loop)
  播放 sentence command，對應 0x80 起算的句子編號。
- playStop/playPause/playContinue/playRepeat
  分別送 0xF8、0xF1、0xF2、0xF4。
- isPlaying()
  讀 STATUS_PIN 判斷 busy/no busy。

C. Key/LED/Power
- scanKey()
  掃 A1-A5 按鍵並更新內部 key value。
- isKeyAction()
  回傳是否有 key action。
- readKeyValue()
  讀目前 key bitmask。
- setPower(status)
  控制 POWER_PIN。
- setLED(status)
  控制 LED_PIN。

D. Voice source update
- initAudioUpdate(baudrate)
  初始化序列埠/更新通道，預設 baudrate 256000。
- isUpdateBegin()
  偵測是否收到更新開始訊號。
- executeUpdate()
  執行語音資料更新流程。
- recAudioData()
  接收音訊資料。
- programEntry、ack、dummyClocks、sendAddr、sendData、readData 等
  用於進入 program mode、傳送位址/資料、確認 ACK。
- switchSPIMode()
  切換到 SPI flash 操作模式。
- SPIFlashChipErase、SPIFlashPageWrite、SPIFlashReadSFDP 等
  操作外部 SPI flash。
- checkCRC8()
  使用 CRC8 table 驗證資料。

掃描時看到的注意事項：
- initAudioUpdate、isUpdateBegin、executeUpdate、recAudioData 在函式清單中各出現兩次，可能是因為檔案內有不同編譯條件或重複實作區塊；若後續要維護，建議檢查是否有條件編譯包住。
- pin 腳位寫死，不像 BMS26M833 可在 constructor 傳 pin，移植性較低。
- BMV31T001_UPDATA_BEGIN 拼字是 UPDATA，不是 UPDATE；這是 library 既有常數名稱，使用時需照原名。

九、整體使用方式

最基本播放流程：

1. include library 與 voice list
   #include "BMV31T001.h"
   #include "voice_cmd_list.h"

2. 建立物件
   BMV31T001 myBMV31T001;

3. 初始化與開電
   myBMV31T001.begin();
   myBMV31T001.setPower(BMV31T001_POWER_ENABLE);

4. 設定音量
   myBMV31T001.setVolume(6);

5. 播放指定語音
   myBMV31T001.playVoice(VOC_1);

6. 停止/暫停/繼續
   myBMV31T001.playStop();
   myBMV31T001.playPause();
   myBMV31T001.playContinue();

7. 判斷是否播放中
   if (myBMV31T001.isPlaying() == BMV31T001_BUSY) { ... }

8. 若要允許語音更新
   myBMV31T001.initAudioUpdate();
   if (myBMV31T001.isUpdateBegin() == BMV31T001_UPDATA_BEGIN) {
     myBMV31T001.executeUpdate();
   }

十一、建議整理事項

1. 確認第 10 組語音檔名是否應為「十」而不是「坋」，若要跨平台使用，建議改成 10.mp3/10.wav/10.COD 等 ASCII 檔名。
2. 將 source 註解統一轉成 UTF-8，避免亂碼。
3. 若專案只要播放已燒錄語音，不需要保留全部 VoiceBroadcast 中間檔；至少需要 library source、voice_cmd_list.h 與 Arduino sketch。
4. 若要重新產生語音或更新模組，需保留 VoiceBroadcast.dat、VoiceBroadcast.ini、Setting.ini、Voice Files 與 output。
5. BMV31T001.cpp 的硬體 pin 是固定值；若要和其他專案整合，建議改成 constructor 可設定 pin，或建立包裝層集中管理腳位。
6. 與 BMC81M001 的 TCP.ino 整合時，通常只會用 begin、setPower、setVolume、initAudioUpdate、playVoice 這幾個 API。
