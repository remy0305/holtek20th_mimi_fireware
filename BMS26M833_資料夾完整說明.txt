掃描日期：2026-04-22

一、總覽

本資料夾實際 library 根目錄為：
BMS26M833/BMS26M833

這是一個 Arduino library，主要用途是透過 I2C 操作 BMS26M833 紅外線熱電堆陣列模組。它可讀取 8 x 8 共 64 點溫度矩陣、讀取 thermistor 溫度、設定中斷門檻，並提供範例將 8 x 8 溫度資料轉成 TFT 熱影像。

資料夾內共有 18 個檔案：
- 1 個 Arduino library metadata：library.properties
- 1 個 Arduino IDE 關鍵字設定：keywords.txt
- 2 個 library 原始碼：BMS26M833.h、BMS26M833.cpp
- 3 個範例 sketch：readTemperature、setInterrupt、DisplayThermalImagingOnTheTFT
- 3 個熱影像/TFT 輔助程式檔：InfraredThermalImaging.c、InfraredThermalImaging.h、lcdfont.h
- 6 張範例圖片：layout.png、schematic.png
- 2 個 Windows 縮圖快取：Thumbs.db

注意：
- 多處註解中的攝氏度符號與中文歷史字串在目前讀取環境下顯示成亂碼，推測是原始編碼或特殊符號造成。程式邏輯本身可正常判讀。
- 此 library 需要 Arduino Wire/I2C。
- DisplayThermalImagingOnTheTFT 範例額外依賴 BMD58T280 TFT library、SPI，以及特定 BMduino/UNO 接線條件。

4. src/BMS26M833.cpp

行數：530
大小：18005 bytes

Library 的主要實作檔，透過 Wire/I2C 讀寫 BMS26M833 暫存器並換算溫度。

主要功能分類：

A. 全域資料
- dataBuff[128]：讀寫 I2C 資料的暫存陣列。
- DataCnt：宣告但掃描中未見實際使用。
- extern float TempMax, TempMin：供 DisplayThermalImagingOnTheTFT/InfraredThermalImaging 相關範例共用最大/最小溫度。

B. 建構與初始化
- BMS26M833(intPin, theWire)
  記錄 INT pin 與 Wire 物件。
- begin(i2c_addr)
  設定 INT pin 為 INPUT、初始化 Wire、進入 NORMAL_MODE、做 INITIAL_RESET、delay 1000、設定 FPS_10、關閉 interrupt。

C. 基礎 I2C register 操作
- writeReg(addr, data)
  寫入單一 register。
- readReg(addr)
  讀取單一 register。
- readReg(addr, rBuf, rLen)
  從指定 register 連續讀多個 byte。
- writeBytes(wbuf, wlen)
  Wire.beginTransmission、write、endTransmission。
- readBytes(rbuf, rlen)
  Wire.requestFrom 後讀入指定長度。

D. 讀取 8 x 8 溫度矩陣
- readPixels(tempBuff)
  從 REG_T01L、REG_T17L、REG_T33L、REG_T49L 分四段各讀 32 bytes，共 128 bytes，轉成 64 個溫度值。
  每個 pixel 由 low/high byte 組成，乘以 0.25 作為溫度解析度。
- readPixelsAndMaximum(tempBuff, maxValue, minValue)
  與 readPixels 類似，但同時計算最大與最小溫度。

E. Thermistor 溫度
- readThermistorTemp()
  讀 REG_TTHL/REG_TTHH，將 12-bit signed value 轉成溫度，解析度 0.0625。
- readRawThermistorTemp()
  回傳 thermistor raw data。

F. Interrupt
- setINT(isEnable)
  true 時寫 REG_INTC = 0xff 啟用中斷，false 時寫 0x00 關閉。
- setInterruptLevels(high, low)
  hysteresis 預設為 high * 0.95。
- setInterruptLevels(high, low, hysteresis)
  將 high/low/hysteresis 轉成 signed 12-bit 格式後寫入 interrupt level registers。
- getINT()
  digitalRead(_intpin)，註解說 0 表示收到 interrupt。
- getINTTable(buf, size)
  從 0x10 開始讀 interrupt table，預設 8 bytes。
- setStatusClear()
  呼叫 reset(FLAG_RESET) 清除 interrupt/status。

G. 模組狀態與模式
- getOperationMode()
- sleep()
- reset(mode)
- getFrameMode()
- getStatus()
- getAverageOutputMode()
- beginMeasure(mode)
- setFrameMode(mode)
- setAverageOutputMode(mode)
- setOperationMode(mode)

掃描時看到的程式注意事項：
- getOperationMode()、getFrameMode()、getStatus()、getAverageOutputMode() 呼叫 readReg(REG_xxx)，但接著讀 dataBuff[0]。readReg(addr) 本身會回傳讀值，雖然目前 readReg() 也會更新 dataBuff[0]，但寫法容易讓人誤解。
- readPixels() 直接把 raw data 轉 uint16_t 再乘 0.25，沒有像 thermistor 那樣處理 signed extension。若 sensor pixel 溫度可能出現負值，需確認 datasheet 是否允許這種解法。
- DataCnt 全域變數目前未見使用。
- setAverageOutputMode() 會連續寫 0x1f 特殊序列 0x50、0x45、0x57，再改 REG_AVE，推測是解鎖 hidden/protected register 的程序；這段應保留但需要文件註解。

六、setInterrupt 範例

9. examples/setInterrupt/setInterrupt.ino

行數：60
大小：2154 bytes

示範如何使用 BMS26M833 的 interrupt 功能。

主要設定：
- TEMP_INT_HIGH = 30
- TEMP_INT_LOW = 15
- BMS26M833 Amg
- interruptTable[8]

setup():
- Serial.begin(9600)
- Amg.begin()
- Amg.setInterruptLevels(TEMP_INT_HIGH, TEMP_INT_LOW)
- Amg.setINT(true)

loop():
- 如果 Amg.getINT() == 0，代表收到 interrupt。
- 呼叫 Amg.getINTTable(interruptTable) 讀出 8 bytes interrupt table。
- 逐 bit 印出哪些 pixel 觸發中斷。
- 呼叫 Amg.setStatusClear() 清除中斷，準備下一次觸發。

用途：
- 設定高/低溫門檻。
- 找出 8 x 8 中哪些 pixel 超過高門檻或低於低門檻。

- 顯示最大/最小溫度。

14. examples/DisplayThermalImagingOnTheTFT/InfraredThermalImaging.c

行數：228
大小：8570 bytes

熱影像轉換演算法實作。

主要流程：
1. TransformGray()
   將 8 x 8 float 溫度矩陣依 TempMax/TempMin 正規化成 0-255 灰階。
2. 若 TempMax - TempMin 小於等於 TempDiff，直接不處理。

   使用 Otsu/最大類間變異法找背景與目標的分割門檻。

5. Bilinear()
   用雙線性插值將 8 x 8 放大到指定輸出尺寸，例如 55 x 55。

主要函式：
- InfraredThermalImaging(ThermalImagingConfig *Config)
- Bilinear(...)
- TransformGray(...)
- BackgroundFiltering(...)
- Otus(...)

注意：
- 檔案中函式註解寫 Otus，實際常見名稱是 Otsu。
- 使用 extern float TempMax, TempMin，與 DisplayThermalImagingOnTheTFT.ino 的全域變數耦合。

八、整體 API 使用方式

最基本讀取：

1. 建立物件
   BMS26M833 Amg;

2. 初始化
   Amg.begin();

3. 讀取 64 點溫度
   float TempMat[64];
   Amg.readPixels(TempMat);

4. 讀取 thermistor
   float temp = Amg.readThermistorTemp();

5. 若需要最大/最小值
   float maxTemp, minTemp;
   Amg.readPixelsAndMaximum(TempMat, maxTemp, minTemp);

6. 若需要 interrupt
   Amg.setInterruptLevels(30, 15);
   Amg.setINT(true);
   if (Amg.getINT() == 0) {
     uint8_t table[8];
     Amg.getINTTable(table);
     Amg.setStatusClear();
   }

十、建議整理事項

1. 將亂碼註解或攝氏度符號統一轉成 UTF-8。
2. 將 Thumbs.db 加入 .gitignore 或刪除。
3. 若專案只需要讀溫度，可只保留 src 與 readTemperature 範例；TFT 熱影像範例依賴較多。
4. DisplayThermalImagingOnTheTFT.ino 的 amg(22, &Wire1) 需要依實際板子調整。
5. 若要整合到 BMC81M001 的 TCP.ino，通常只需要 BMS26M833.h/cpp 與 readPixels()/readThermistorTemp() 這兩個 API。
6. 可補一份 README.md，因為此資料夾目前沒有 README，使用者只能從範例和 library.properties 理解功能。
